# Controller Feature — Scale, Rotate & AR Passthrough

## 1. Add scale & rotation fields to WallElement (all element types)

### DB Changes — `elements/basic.py`
```
WallElement base class adds:
  scale_x = FloatField(default=1.0)
  scale_y = FloatField(default=1.0)
  scale_z = FloatField(default=1.0)
  rotation_x = FloatField(default=0.0)
  rotation_y = FloatField(default=0.0)
  rotation_z = FloatField(default=0.0)
```

### to_aframe() changes
Each element type already has scale/rotation — just change to use DB fields:
- **Image**: add `scale` and `rotation` attributes using scale_x,y,z and rotation_x,y,z
- **Webpage**: add `scale` and `rotation` attributes  
- **GaussianSplat**: already has scale/rotation strings — instead use DB float fields
- **GLTFmodel**: already has scale/rotation strings — instead use DB float fields

## 3. PATCH endpoint update

Support new fields:
```
scale_x, scale_y, scale_z
rotation_x, rotation_y, rotation_z
```

All 6 values + 2 position values saved in one PATCH call.

## 4. AR Passthrough (Admin only)

Detection: `navigator.xr.isSessionSupported('immersive-ar')`

If supported, show "🥽 AR Passthrough" button in admin panel.
On click → request AR session with passthrough enabled.
Background shows real camera feed, virtual elements overlaid.

A-Frame: `a-scene` with `renderer="colorManagement: true; passthrough: true"` + WebXR AR session.
